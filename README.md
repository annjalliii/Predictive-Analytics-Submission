# Diabetes Hospital Readmission Prediction

**MSIN0097 Predictive Analytics — Individual Coursework**
MSc Business Analytics | UCL School of Management | March 2026

## Project Overview

Binary classification system to predict whether a diabetic patient will be readmitted to hospital within 30 days of discharge, using encounter-level clinical data from 130 US hospitals.

**Dataset:** [Diabetes 130-US Hospitals (1999–2008)](https://archive.ics.uci.edu/dataset/296) — 101,766 encounters × 50 features

**Final Model:** Tuned XGBoost — AUC-ROC: 0.655 | Recall: 0.608 | F1: 0.261

## Methodology

1. **Data Cleaning:** Removed deceased patients, high-missingness columns (weight 97%, medical_specialty 49%, payer_code 40%), grouped 700+ ICD-9 codes into 9 clinical categories, engineered medication summary features.

2. **Preprocessing:** Stratified 80/20 split → median imputation + StandardScaler (numerical) → mode imputation + one-hot encoding (categorical) → SMOTE on training only. All transformations fitted on training data exclusively to prevent leakage.

3. **Modelling:** Four models trained (Logistic Regression, Random Forest, XGBoost, MLP). 5-fold stratified cross-validation with SMOTE inside each fold confirmed robustness. Ablation study quantified SMOTE and engineered feature contributions.

4. **Evaluation:** Tuned XGBoost selected. Threshold optimisation analysis conducted. Error analysis revealed missed readmissions had fewer prior visits and shorter stays.

## Agent Tooling

This project was developed with the assistance of Claude (Anthropic) as a collaborative tool. Four critical agent errors were identified and corrected:

1. Accuracy suggested as primary metric (rejected — misleading for 89:11 imbalance)
2. StandardScaler applied before train-test split (data leakage — corrected)
3. SMOTE applied before split (synthetic leakage — corrected)
4. Cross-validation on SMOTE-resampled data (inflated scores — corrected to use imblearn.Pipeline)

Full Decision Register and Agent Usage Log are in the submission report appendix. Chat transcripts are in `docs/agent_logs/`.
